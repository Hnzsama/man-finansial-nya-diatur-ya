# Architecture Rules

Sebelum menulis kode:

1. Analisis struktur proyek.
2. Jangan mengganti stack.
3. Jangan menambahkan dependency tanpa alasan.
4. Ikuti pola yang sudah ada.

---

## Prinsip

- KISS
- SOLID
- DRY

---

## Larangan

Jangan:

- overengineering
- membuat service yang tidak diperlukan
- membuat abstraction terlalu dini
- membuat generic repository

---

## Prioritas

1. Maintainability
2. Readability
3. Performance
4. Scalability
